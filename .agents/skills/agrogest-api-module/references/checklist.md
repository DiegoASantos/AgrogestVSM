# Checklist de módulo API

- Seguir `application/`, `infrastructure/`, `presentation/` y módulo vecino.
- Validar DTOs, IDs, paginación y filtros.
- Documentar respuestas y errores en Swagger.
- Traducir constraints a excepciones útiles.
- Proteger endpoints y exigir roles en API.
- Evitar secretos y PII en logs.
- Probar reglas, autorización y casos límite.
- Ejecutar lint, typecheck, tests y build de API.
