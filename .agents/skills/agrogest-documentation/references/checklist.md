# Checklist documental

- Clasificar en arquitectura, dominio, ADR, spec, runbook, operación o nota.
- Actualizar el documento activo en vez de duplicarlo.
- Contrastar con código, pruebas, configuración y migraciones.
- Mantener metadatos y enlazar desde `docs/index.md`.
- Actualizar documentación vigente al implementar una spec.
- Eliminar secretos, datos reales y ejemplos sensibles.
- Comprobar enlaces, rutas, comandos y `git diff --check`.
- Registrar riesgos y deuda restantes.
